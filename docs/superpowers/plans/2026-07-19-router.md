# Router (RAG + KEV Correlation) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Correlate the RAG lane's articles with the KEV lane's confirmed-exploited catalog by CVE ID, surfacing dual-confirmed threats first, and produce a standalone `unified_digest.md`, refreshed automatically whenever either source lane updates.

**Architecture:** A small additive change to the existing KEV lane (emit structured `kev_data.json` alongside its markdown), plus one new pure-Python correlation module (`router.py`) that reads both lanes' structured data (`audit.jsonl` + `kev_data.json`) and renders a three-section unified digest — no LLM calls, no changes to either lane's own analysis/ranking logic.

**Tech Stack:** Python 3.11+ standard library only (`json`, `re`, `dataclasses`) — no new dependencies. pytest for tests.

## Global Constraints

- `retrieved_chunks` in `audit.jsonl` is a list of `[chunk_text, score]` two-element lists (verified against a real record) — chunk text is index `0` of each pair
- `prompt` in `audit.jsonl` always contains the fixed phrase `document titled "..."` (from `llm_analysis.build_prompt`'s template) — title extraction relies on this exact phrase
- Only `stage == "analysis"` records from `audit.jsonl` are used for correlation — `stage == "synthesis"` records are ignored
- CVE ID pattern: `CVE-\d{4}-\d{4,7}` (matches CISA's own ID format)
- RAG-only high-risk threshold: `risk_score >= 6` (same value as `pipeline.py`'s existing `SYNTHESIS_THRESHOLD` — router defines its own constant, does not import across modules)
- KEV-only section is capped at 30 entries: remove any CVE IDs already matched in the dual-confirmed section, then re-sort the remainder by `date_added` descending (recency), then cap. This deliberately does NOT reuse `kev_data.json`'s ransomware-first order — ransomware-linked entries are only ~9% of the full 200 (18/200), so capping on top of ransomware-first order would make this section ~60% ransomware, a large artificial skew. Recency ordering keeps the section representative of the real KEV composition; each entry still carries its own `(RANSOMWARE-LINKED)` flag inline.
- No LLM calls, no new pip dependencies anywhere in this plan
- No placeholders, no speculative error handling beyond what's specified — YAGNI

- [ ] **Step 3: Implement `router.py`**

```python
import json
import re
from pathlib import Path

RISK_THRESHOLD = 6
KEV_ONLY_CAP = 30

CVE_PATTERN = re.compile(r"CVE-\d{4}-\d{4,7}")
TITLE_PATTERN = re.compile(r'document titled "(.*?)"')


def extract_cve_ids(text: str) -> set[str]:
    return set(CVE_PATTERN.findall(text))


def extract_title(prompt: str) -> str:
    match = TITLE_PATTERN.search(prompt)
    if not match:
        raise ValueError("could not extract title from prompt")
    return match.group(1)


def load_analysis_records(audit_path: Path) -> list[dict]:
    records = []
    with open(audit_path, "r", encoding="utf-8") as f:
        for line in f:
            line = line.strip()
            if not line:
                continue
            record = json.loads(line)
            if record.get("stage") == "analysis":
                records.append(record)
    return records


def load_kev_entries(kev_data_path: Path) -> list[dict]:
    with open(kev_data_path, "r", encoding="utf-8") as f:
        return json.load(f)


def build_unified_digest(audit_path: Path, kev_data_path: Path) -> str:
    analysis_records = load_analysis_records(audit_path)
    kev_entries = load_kev_entries(kev_data_path)
    kev_by_cve = {entry["cve_id"]: entry for entry in kev_entries}

    dual_confirmed = []
    rag_only = []
    matched_cve_ids = set()

    for record in analysis_records:
        title = extract_title(record["prompt"])
        chunk_text = " ".join(chunk[0] for chunk in record["retrieved_chunks"])
        cve_ids = extract_cve_ids(chunk_text)
        risk_score = record["risk_score"]

        matched_entries = [kev_by_cve[cve_id] for cve_id in cve_ids if cve_id in kev_by_cve]
        if matched_entries:
            matched_cve_ids.update(entry["cve_id"] for entry in matched_entries)
            dual_confirmed.append((title, risk_score, matched_entries))
        elif risk_score >= RISK_THRESHOLD:
            rag_only.append((title, risk_score))

    unmatched_kev = [
        entry for entry in kev_entries if entry["cve_id"] not in matched_cve_ids
    ]
    # Re-sort by recency (not the ransomware-first order kev_data.json
    # arrives in) so a hard cap doesn't overrepresent the ~9% of entries
    # that are ransomware-linked -- see design spec rationale.
    kev_only = sorted(
        unmatched_kev, key=lambda entry: entry["date_added"], reverse=True
    )[:KEV_ONLY_CAP]

    lines = ["# Unified Threat Digest", ""]

    lines.append("## Dual-Confirmed (RAG article + CISA KEV)")
    lines.append("")
    if dual_confirmed:
        for title, risk_score, matched_entries in dual_confirmed:
            for entry in matched_entries:
                lines.append(
                    f"### {entry['cve_id']} - {title} (risk score: {risk_score}/10)"
                )
                lines.append(
                    f"**Vendor/Product:** {entry['vendor_project']} {entry['product']}"
                )
                lines.append(f"**Vulnerability:** {entry['vulnerability_name']}")
                lines.append("")
    else:
        lines.append("No dual-confirmed items in this run.")
        lines.append("")

    lines.append("## RAG-Only High-Risk (not in CISA KEV)")
    lines.append("")
    if rag_only:
        for title, risk_score in rag_only:
            lines.append(f"### {title} (risk score: {risk_score}/10)")
            lines.append("")
    else:
        lines.append("No RAG-only high-risk items in this run.")
        lines.append("")

    lines.append("## KEV-Only (not covered by RAG lane)")
    lines.append("")
    if kev_only:
        for entry in kev_only:
            ransomware_note = (
                " (RANSOMWARE-LINKED)" if entry["known_ransomware_use"] else ""
            )
            lines.append(
                f"### {entry['cve_id']} - {entry['vendor_project']} "
                f"{entry['product']}{ransomware_note}"
            )
            lines.append(f"**Vulnerability:** {entry['vulnerability_name']}")
            lines.append("")
    else:
        lines.append("No KEV-only items in this run.")
        lines.append("")

    return "\n".join(lines)


if __name__ == "__main__":
    digest_markdown = build_unified_digest(Path("audit.jsonl"), Path("kev_data.json"))
    with open("unified_digest.md", "w", encoding="utf-8") as f:
        f.write(digest_markdown)
    print("Wrote unified_digest.md")
```

- [ ] **Step 1: Create `.github/workflows/router.yml`**

```yaml
name: Build unified digest

on:
  push:
    branches: [master]
    paths:
      - 'digest.md'
      - 'audit.jsonl'
      - 'kev_data.json'
  workflow_dispatch: {}

jobs:
  route:
    runs-on: ubuntu-latest
    permissions:
      contents: write
    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-python@v5
        with:
          python-version: '3.11'

      - name: Build unified digest
        run: PYTHONPATH=src python -m threat_digest.router

      - name: Commit unified digest
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "github-actions[bot]@users.noreply.github.com"
          if [ -n "$(git status --porcelain)" ]; then
            git add unified_digest.md
            git commit -m "chore: refresh unified digest"
            git push
          else
            echo "No changes to commit."
          fi
```

The `paths` filter means this workflow only runs when one of the three source files actually changes on `master` — not on every push to the repo. `permissions: contents: write` is required for the default `GITHUB_TOKEN` to push, same as `kev_fetch.yml`.
